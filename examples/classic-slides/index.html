在Stitch Fix模式中
技术和算法起到什么作用？


客户填写资料 - 寄送5件衣物 - 客户为选择付款


客户填写个人资料



根据客户填写的个人资料和需求
自动生成一个预约发货订单

分配库存

根据用户收货地点仓库库存匹配情况，计算出货仓库

选衣服

第一步：用不同的智能算法对衣物按不同纬度排序

第二步：从矩阵中去除推荐过的衣物

第三步：通过协同过滤算法选择


第四步：当属性明确时，用混合效果模型进行选择

第五步：当属性不明确时，根据反馈和购买历史推测


第六步：根据反馈文本做语义分析，以完善个人资料

第七步：根据Pinterest的关注或者提交的图片
做图像识别和分析，以完善个人资料

第八步：所有以上的算法，只是为了给衣服打分
为理型师选取衣服做参考

利用算法给用户分配适合的理型师


  • 等待时间
  • 客户的关系
  • 风格和客户是否契合

理型师亲手为客户挑选5件衣服


  • 算法学习理型师怎么做决定
  • 帮助理型师更快做决定

用户试衣服并填写反馈


  • 通过一次次反馈，记录客户的 变化
  • 用马尔可夫模型来 预测 客户未来的可能需求

根据数据预测库存

基于数据和算法的服饰设计


  • 每个样式点作为『基因』
  • 所有的衣服的所有样式点，构成一个基因集合
  • 重新组合得分高的基因，加入突变基因
  • 设计师可以从中精炼出适合的服装组合

日播盒子的技术架构


电话联系用户 - 挑选5件衣服 - 客户为选择付款


数据录入(后台)


  • 创建产品库
  • 录入服装元素向量 (clothing vector)
  • 用户数据的导入及编辑更新

理型师选择衣服(后台)


  • 查看会员信息
  • 查看初步匹配到的衣服
  • 为用户选择5件衣服，填写搭配建议
  • 打印仓库派送单 和 盒子订单 (包含支付二维码)

仓管更新订单(后台)


  • 填写订单派送快递单号和返回单号
  • 收到寄回衣服，更新订单状态

用户客户端(小程序)


  • 用户扫码支付
  • 查看订单内容
  • 填写反馈
  • 修改个人信息

算法


  • 根据现有的会员数据，通过数据算法分析优质会员
  • 创建产品库以及服装特征向量 (clothing vector)
  • 研发产品各种排序算法
  • 研发产品匹配算法

系统架构

整个技术栈采用微服务架构



  • 系统易于更新维护
  • 满足技术多样性

用户前端使用微信小程序


  • 跨平台，无需按照APP
  • 方便在微信生态内推广

用户服务器端
Ruby on Rails + Restful API



  • 高效开发，快速迭代
  • 硅谷创业首选框架

后台前端使用React
蚂蚁金服的Ant Design


  • 组件化框架开发
  • 易于维护和扩展
  • UI界面美观，标准化

后台服务器端/算法实现
Ruby on Rails / Java / nodejs


  • Stitch Fix开源项目中大量实用Ruby，有可以借鉴的工具
  • 在需要性能支持的服务中，可以使用Java，保证性能要求
